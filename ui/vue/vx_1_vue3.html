# #open in new window非兼容的变更

下面列出了从 2.x 开始的非兼容的变更：

# #open in new window全局 API

  • 全局 Vue API 已更改为使用应用程序实例open in new window
  • 全局和内部 API 已经被重构为支持 tree-shakeopen in new window

# #open in new window模板指令

  • 组件上 v-model 用法已更改，以替换 v-bind.syncopen in new window
  • ` 和非 `v-for` 节点上的 `key` 用法已更改open in new window
  • 在同一元素上使用的 v-if 和 v-for 优先级已更改open in new window
  • v-bind="object" 现在排序敏感open in new window
  • v-on:event.native 修饰符已移除open in new window
  • v-for 中的 ref 不再注册 ref 数组open in new window

# #open in new window组件

  • 只能使用普通函数创建函数式组件open in new window
  • `functional` attribute 在单文件组件 (SFC) 的 ` 和 `functional` 组件选项中被废弃open in new window
  • 异步组件现在需要使用 defineAsyncComponent 方法来创建open in new window
  • 组件事件现在需要在 emits 选项中声明open in new window

# #open in new window渲染函数

  • 渲染函数 API 更改open in new window
  • $scopedSlots property 已移除，所有插槽都通过 $slots 作为函数暴露open in new window
  • $listeners 被移除或整合到 $attrsopen in new window
  • $attrs 现在包含 class 和 style attributeopen in new window

# #open in new window自定义元素

  • 自定义元素检测现在在模板编译时执行open in new window
  • 特殊的 `is` attribute 的使用被严格限制在被保留的 `` 标签中open in new window

# #open in new window其他小改变

  • destroyed 生命周期选项被重命名为 unmounted
  • beforeDestroy 生命周期选项被重命名为 beforeUnmount
  • default prop 工厂函数不再可以访问 this 上下文open in new window
  • 自定义指令的 API 已更改为与组件生命周期一致，且 binding.expression 已移除open in new window
  • data 选项应始终被声明为一个函数open in new window
  • 来自 mixin 的 data 选项现在为浅合并open in new window
  • Attribute 强制策略已更改open in new window
  • 一些过渡的 class 被重命名open in new window
  • `` 不再默认渲染包裹元素open in new window
  • 当侦听一个数组时，只有当数组被替换时，回调才会触发，如果需要在变更时触发，则必须指定 deep 选项open in new window
  • 没有特殊指令的标记 (v-if/else-if/else、v-for 或 v-slot) 的 现在被视为普通元素，并将渲染为原生的 元素，而不是渲染其内部内容。
  • 已挂载的应用不会取代它所挂载的元素open in new window
  • 生命周期的 hook: 事件前缀改为 vnode-open in new window

# #open in new window被移除的 API

  • keyCode 作为 v-on 修饰符的支持open in new window
  • $on、$off 和 $once 实例方法open in new window
  • 过滤器 (filter)open in new window
  • 内联模板 attributeopen in new window
  • $children 实例 propertyopen in new window
  • propsData 选项open in new window
  • $destroy 实例方法。用户不应再手动管理单个 Vue 组件的生命周期。
  • 全局函数 set 和 delete 以及实例方法 $set 和 $delete。基于代理的变化检测已经不再需要它们了。
